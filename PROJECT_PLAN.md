# كافيه مزاج — خطة المشروع وتوثيقه

آخر تحديث: 2026-07-05

**مستودع GitHub:** https://github.com/muqa997/mazaaj

## الفكرة العامة

موقع ويب فاخر وتفاعلي (وليس تطبيق موبايل) لكافيه مزاج، يخدم عدة أهداف:

1. **عرض هوية الكافيه**: القصة، الأجواء، الطابق الثاني (صالة الألعاب).
2. **منيو تفاعلي بصفحة واحدة** (`/menu`): يُستخدم كوجهة لرابط QR الموضوع على طاولات الكافيه، ليتصفح الزبون المنتجات من جواله بدل المنيو الورقي.
3. **طلب توصيل** (قهوة، حلويات، أراجيل): بدون أي نظام حسابات مستخدمين — سلة بسيطة، ثم نموذج بيانات، ثم إرسال الطلب كرسالة جاهزة عبر واتساب ليأكده فريق الكافيه يدوياً.
4. **عرض خدمات الكافيه، موقعه على الخريطة، وروابط التواصل الاجتماعي**.
5. **وضع داكن (Dark Mode)** كامل، ودعم لغتين (عربي/إنجليزي).
6. **لوحة تحكم خاصة بالمدراء** (مسار سري، غير مرتبط من الموقع نهائياً) لعرض الطلبات والإحصائيات والمتقدمين للوظائف وإدارة الوظائف الشاغرة.

الموقع **Mobile-First** (95% آيفون)، لكنه مصمم كموقع ويب حقيقي (Navbar + سكرول بين أقسام) وليس بواجهة تطبيق (لا يوجد Bottom Tab Bar).

## القرارات المعمارية المتفق عليها

| القرار | الخيار المُعتمد | السبب |
|---|---|---|
| هيكلة الموقع | صفحة رئيسية طويلة بالسكرول (أقسام) + صفحات مستقلة لـ `/menu` و `/order` | يحس الزائر إنه موقع تفاعلي حقيقي، ويسهّل ربط QR بصفحة منيو منفصلة |
| اللغتان | next-intl مع رابط لغة صريح (`/ar`, `/en`)، والعربية دائماً الافتراضية (بدون كشف تلقائي حسب المتصفح) | أفضل لمحركات البحث ومشاركة الروابط |
| نظام الطلبات | بدون تسجيل دخول/حسابات للزبون — سلة محلية (localStorage) + نموذج بيانات + تسليم الطلب كرسالة واتساب جاهزة، **وحفظ نسخة من الطلب بجدول `orders` بـ Supabase** لأغراض السجل والإحصائيات بلوحة التحكم | الكافيه ما يحتاج نظام باك-إند معقد أو بوابة دفع، لكن يحتاج يشوف سجل الطلبات وإحصائياتها |
| لوحة التحكم | مسار سري بأحرف وأرقام عشوائية (غير مرتبط من أي مكان بالموقع) + رمز دخول مشترك لكل المدراء (أي نص تختاره أنت، بأي طول)، جلسة موقّعة (HMAC) صالحة 3 أيام وتتجدد تلقائياً مع الاستخدام | حماية بسيطة وكافية لأداة داخلية بدون حاجة لنظام مستخدمين كامل |
| الوضع الداكن | متغيرات CSS (`--color-background/primary/accent`) تتبدل مع صنف `.dark`، عبر مكتبة `next-themes` | يخلي كل الموقع ينقلب تلقائياً بدون تكرار كود بكل مكوّن |
| قسم "من نحن" | داخل الفوتر نفسه (مو قسم مستقل بمنتصف الصفحة) — لوجو دائري فوق فقرة تعريفية | طلب المستخدم إنه يكون بمكان "نبذة عنا" الموجود أصلاً بالفوتر |
| هيدر الجوال | الهمبركر والسلة يتبدلون جهتهم تلقائياً حسب اتجاه اللغة (RTL/LTR) عبر flexbox عادي بدون أي شرط برمجي، واللوجو دايماً بالنص | يطابق الاتجاه الطبيعي للغة بدون كود إضافي |

## إعداد Supabase

الحساب مربوط الحين (الرابط والمفاتيح موجودة بملفك المحلي `.env.local`، وما راح تترفع على GitHub أبداً).

الخطوة المتبقية الوحيدة: افتح **SQL Editor** بمشروعك على Supabase، والصق فيه محتوى ملف [supabase/schema.sql](supabase/schema.sql) كامل، وشغّله مرة وحدة. هذا الملف ينشئ الجداول الثلاثة (`job_applications`, `job_openings`, `orders`) بكل صلاحياتها، ويضيف الوظيفة الحالية (موظف صالة بلياردو) كأول صف، وينشئ bucket تخزين السير الذاتية (`cvs`) — كل شي بضغطة وحدة.

لحد ما تشغّل هذا الملف، صفحة `/careers` تعرض بيانات احتياطية مؤقتة (من [lib/job-openings.ts](lib/job-openings.ts)) عشان الموقع يبقى يشتغل ويبين صحيح.

## لوحة التحكم (Admin Dashboard)

لوحة التحكم جاهزة بالكامل بالكود، ومحمية بثلاث طبقات:

1. **مسار سري** بأحرف وأرقام عشوائية (10 خانات) — غير مكتوب بأي مكان بالكود المرفوع على GitHub، ولا مرتبط أو معروض بأي صفحة بالموقع العام. القيمة نفسها موجودة بس بملفك المحلي `.env.local` (متغير `ADMIN_ROUTE`).
2. **رمز دخول** (متغير `ADMIN_ACCESS_CODE`) — أي نص تختاره أنت (أرقام، حروف، أو مزيج، وبأي طول تقريباً)، رمز واحد مشترك لكل المدراء والمشرفين حالياً.
3. **جلسة موقّعة تلقائياً** (متغير `ADMIN_SESSION_SECRET`) — صالحة 3 أيام من آخر استخدام، وتتجدد تلقائياً كل ما تزور اللوحة؛ إذا مرت 3 أيام بدون زيارة، تحتاج تدخل الرمز مرة ثانية.

**القيم الفعلية لهذي المتغيرات الثلاثة موجودة الحين بملفك المحلي `.env.local`** — راح أعطيك الرابط والرمز بالمحادثة مباشرة، مو هنا بالملف، عشان الملف هذا يترفع على GitHub وأي شخص يقدر يشوفه.

⚠️ **تقدر تغيّر رمز الدخول لأي شي تحبه** — رقمي، بالحروف، أو مزيج، وبأي طول — بس افتح `.env.local` وغيّر قيمة `ADMIN_ACCESS_CODE` وأعد تشغيل السيرفر. لا تحتاج تعديل أي كود.

اللوحة تعرض:
- **نظرة عامة**: عدد الطلبات، إجمالي المبيعات، عدد المتقدمين للوظائف، عدد الوظائف الشاغرة حالياً.
- **الطلبات**: كل طلب توصيل وصل (الاسم، الهاتف، العنوان، تفاصيل الطلب، الإجمالي).
- **المتقدمون**: كل شخص قدّم على وظيفة (الاسم، الهاتف، الوظيفة، الملاحظات، رابط CV إذا رفعه).
- **الوظائف**: إضافة/تعديل/حذف/تفعيل أو إخفاء أي وظيفة — تنعكس مباشرة على صفحة `/careers` العامة.

## ملاحظة بخصوص الهيدر والفوتر

فحصت كل صفحات الموقع (الرئيسية، المنيو، الطلب فارغ/ممتلئ، الوظائف) بالعربي والإنجليزي، والهيدر والفوتر يظهرون بشكل صحيح بكل صفحة — الـ layout المشترك (`app/(site)/[locale]/layout.tsx`) يغلف كل الصفحات دائماً. لاحظت حالة واحدة نادرة أثناء الفحص (تنقل غير متوقع بصفحة الطلب) ما قدرت أكررها بثبات، فأضفت ملف `app/(site)/[locale]/error.tsx` كإجراء احترازي: إذا صار أي خطأ داخل محتوى صفحة معينة مستقبلاً، بيتم احتواؤه داخل تلك الصفحة فقط مع زر "إعادة المحاولة"، والهيدر والفوتر يبقون ظاهرين دائماً حواليه بدل ما يختفوا. إذا شفت الحالة تصير عندك مرة ثانية على جهازك، صوّرها لي بالضبط وين صارت وأتابعها.

## ملاحظة بخصوص اللوجو والوضع الداكن

اللوجو الشفاف الحالي (`mazaaj-logo-trimmed.png`) لونه بني، فيبين بشكل ممتاز بالوضع الفاتح، لكن يصير أخف وضوحاً بالوضع الداكن (تباين أقل مع الخلفية الغامقة). إذا حبيت أزيد وضوحه بالوضع الداكن تحديداً (مثلاً بإضافة دائرة خلفية فاتحة خلفه بذاك الوضع بس)، خبرني وأسويها.
